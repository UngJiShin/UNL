# UNL — UNewLanguage

> 한글 표음문자 체계 기반 다국어 에듀테크 플랫폼

UNewLanguage는 임의의 영어·중국어 텍스트를 한글 표음문자로 자동 변환하고, 에빙하우스 망각곡선 기반 스마트 복습 카드를 자동 생성하는 언어 학습 플랫폼입니다.

## 1. 한글 표음문자 표기법 시스템

외국어 발음을 한글로 표기해 조음 장벽을 낮추는 접근법입니다. 현행 외래어 표기법(국립국어원)의 조음 구별 불가능성을 보완하고, 특허 분쟁(KR20130122437A)을 원천 회피하기 위해 두 가지 독자 규칙을 이원화합니다.

- **옛한글 자모 규칙** — 훈민정음 창제 당시 소실된 자모(ㆄ, ㅸ, ᄙ 등)를 직접 매핑
- **웹 안전식 표준 표음 규칙** — 범용 웹 폰트만으로 표현 가능한 조합 표기

### 1.1 영어 → 한글 표음문자 대응표

| 영어 음소 | 현행 외래어 표기법 | 특허(한글로영어) | UNL 옛한글 규칙 | UNL 웹 안전 규칙 | 조음 원리 |
|-----------|-------------------|----------------|----------------|----------------|-----------|
| /f/ (fun) | ㅍ | ㅍ + 구별기호 | ᅗ | ㅇㅍ | 윗니→아랫입술 마찰, 무성음 |
| /p/ (pen) | ㅍ | ㅍ | ㅍ | ㅍ | 입술 파열, 무성 파열음 |
| /v/ (vine) | ㅂ | ㅂ + 구별기호 | ㅸ | ㅇㅂ | 윗니로 아랫입술 물기, 유성 마찰음 |
| /b/ (best) | ㅂ | ㅂ | ㅂ | ㅂ | 두 입술 파열, 유성 파열음 |
| /l/ (laugh) | ㄹ | ㄹ + 구별기호 | ᄙ | ㄹㄹ | 혀끝→윗잇몸, 설측음 |
| /r/ (run) | ㄹ | ㄹ | ㄹ | ㄹ | 혀 말기, 통과음 |
| /θ/ (think) | ㅅ / ㅌ | ㄷ + 구별기호 | ᅊ | ㄸ | 이 사이 혀끝, 무성 마찰음 |
| /ð/ (this) | ㄷ | ㄷ + 구별기호 | ᅂ | ㅇㄷ | 이 사이 혀끝, 유성 마찰음 |

### 1.2 중국어 → 한글 표음문자 및 성조 시각화

권설음(zh, ch, sh, r)과 설면음(j, q, x)의 조음 위치를 정밀 분할 매핑하고, 성조를 시각 기호로 표시합니다.

| 병음 | IPA | 한글 표준 | UNL 특화 규칙 |
|------|-----|----------|--------------|
| j / q | [t͡ɕ] / [t͡ɕʰ] | ㅈ / ㅊ | ㅈ (지) / ㅊ (치) |
| zh / ch | [ʈ͡ʂ] / [ʈ͡ʂʰ] | ㅈ / ㅊ | ㅈ {즈} / ㅊ {츠} |
| ü / üe | [y] / [y̯œ] | 위 / 웨 | 위 / 웨 |
| ian / uan | [i̯ɛn] / [u̯an] | 옌 / 완 | 옌 / 완 |

**성조 시각화 디자인**

| 성조 | 유형 | 시각화 기호 |
|------|------|------------|
| 1성 | 고평조 | 음절 상단 수평 대시(̄) |
| 2성 | 중고상승조 | 음절 상단 우상향 사선(́) |
| 3성 | 저하강상승조 | 음절 상단 V자형 곡선(̌) |
| 4성 | 고하강조 | 음절 상단 우하향 사선(̀) |
| 경성 | 단촉조 | 기호 생략, 폰트 투명도 50% 감쇄 |

### 1.3 HTML Ruby 태그 아키텍처

옛한글 조합형 렌더링(U+1100~U+11FF)은 모바일 브라우저에서 글자 깨짐을 유발할 수 있습니다. 이를 방지하기 위해 HTML5 `<ruby>` 태그와 CSS Ruby Layout 모듈을 핵심 구조로 채택합니다.

```html
<ruby>
  fun<rp>(</rp><rt>ᅗᅥᆫ</rt><rp>)</rp>
</ruby>
```

- 원문 텍스트는 기저 레이어로 항상 유지 → 폰트 미지원 환경에서도 원문 열람 가능
- 한글 표음 레이어는 루비 텍스트로 오버레이 → 뷰포트 크기에 무관하게 레이아웃 유지

## 2. SRS 알고리즘 (SuperMemo-2)

### 2.1 에빙하우스 망각곡선

사용자가 학습한 카드는 아래 기억 보존 방정식에 따라 복습 시점을 자동 결정합니다.

$$R = e^{-t/S}$$

- $R$: 기억 보존 확률
- $t$: 마지막 인출 후 경과 시간
- $S$: 기억 안정성 (복습 성공 시 기하급수적으로 증가)

| 학습 후 경과 | 망각률 | 보존율 | 플랫폼 개입 시점 |
|-------------|--------|--------|----------------|
| 20분 | 42% | 58% | 1차 활성 인출 자극 |
| 1시간 | 56% | 44% | 임계 시점 알림 |
| 9시간 | 64% | 36% | 당일 종합 복습 권장 |
| 1일 | 67% | 33% | 장기 기억 정착 분수령 |
| 6일 | 75% | 25% | 주간 누적 검토 |
| 31일 | 79% | 21% | 영구 장기 기억 전이 검증 |

### 2.2 SM-2 알고리즘

각 카드는 반복 계수 $n$, 이지 팩터 $EF$ (초기값 2.5), 복습 인터벌 $I$ (일)를 보유합니다.

**인터벌 산출 공식**

$$I(n) = \begin{cases} 1 & n = 1 \\ 6 & n = 2 \\ \text{round}(I(n-1) \times EF) & n > 2 \end{cases}$$

**이지 팩터 갱신 공식**

$$EF' = \max\left(1.3,\ EF + \left(0.1 - (5-q)\times(0.08 + (5-q)\times 0.02)\right)\right)$$

**자가 평가 점수 정의**

| 점수 | 상태 | EF 변동 | 스케줄러 동작 |
|------|------|---------|--------------|
| 5 (Perfect) | 즉각·정확 조음 | +0.10 | 최장 주기로 가속 |
| 4 (Good) | 약간 망설임 후 정답 | 0.00 | EF 유지, 등비 성장 |
| 3 (Pass) | 간신히 인출 | −0.14 | 성공이나 EF 삭감 |
| 2 (Hard) | 인출 실패, 확인 후 인지 | −0.32 | n=0 초기화, 내일 복습 |
| 1 (Bad) | 조음 불가 | −0.54 | n=0 초기화, 당일 반복 |
| 0 (Blackout) | 일체 연상 불가 | −0.80 | 완전 신규 카드 리셋 |

## 3. Gemini TTS API 아키텍처

### 3.1 모델별 비용 비교

| 모델 | 입력 토큰 (1M) | 출력 오디오 (1M) | 1시간 오디오 단가 | 배치 전략 |
|------|--------------|----------------|-----------------|-----------|
| Gemini 2.5 Flash TTS | $0.15 | $6.00 | ~$0.54 | 무료 유저 기본 |
| Gemini 3.1 Flash TTS | $1.00 | $20.00 | ~$1.80 | Pro 유저 선택 |
| Gemini 2.5 Pro TTS | $1.25 | $20.00 | ~$1.80 | 고품질 특화 |

**10분 오디오 (8K 입력 / 15K 출력) 단가 비교**

- Flash 2.5: $8{,}000 \times \frac{\$0.15}{1M} + 15{,}000 \times \frac{\$6.00}{1M} = \mathbf{\$0.0912}$
- Flash 3.1: $8{,}000 \times \frac{\$1.00}{1M} + 15{,}000 \times \frac{\$20.00}{1M} = \mathbf{\$0.308}$ (약 237% 비용 절감)

### 3.2 비용 최적화 아키텍처

**Batch API 적용**
- 실시간이 불필요한 연산(야간 복습 카드 사전 생성 등)은 Batch API로 처리 → 표준 단가 50% 절감
- Flash 2.5 Batch 기준: 입력 $0.075/1M, 출력 오디오 $3.00/1M

**캐싱 레이어**
- 텍스트 문자열을 MD5 해시로 색인 → 기존 캐시 히트 시 API 호출 차단
- Redis (메모리 캐시) + AWS CloudFront CDN (엣지 배포) 이중 레이어
- 사전 생성된 음성은 AWS S3에 적재 후 초고속 서빙

## 4. 수익화 모델

### 4.1 바이럴 중심 무상 공유 모델

변환한 한글 표음 카드 세트를 유니크 URL로 추출해 SNS·커뮤니티에 무료 배포 가능 → 가입자 풀을 유기적으로 확보합니다.

### 4.2 단계별 요금제

| 등급 | 요금 | 주요 기능 | API 품질 | KPI |
|------|------|---------|---------|-----|
| Free | $0.00 / 영구 | 일일 변환 15회, 공개 카드 임포트 | Gemini 2.5 Flash TTS | 신규 유저 유치, 바이럴 |
| Pro | $9.99 / 월 | 무제한 변환·업로드, 개인 저장소 | Gemini 3.1 Flash HQ (24kHz) | MRR, 리텐션 |
| Enterprise | 원생 수 비례 슬라이딩 | 진도 관리 콘솔, 인쇄 PDF 출력 | 단체 배치 파이프라인 | ARR, B2B 제휴 |

> **B2B 전략**: 오프라인 어학원이 수기로 제작하던 한글 표음 교재를 3초 만에 PDF로 출력하도록 지원 → 연간 고정 사용료(ARR) 확보

## 5. 고도화 기능 로드맵

### 5.1 양방향 음성 정밀도 진단

학습자가 마이크로 발음하면 Chirp / Gemini 오디오 분석 모델이 원어민 음원과 음소별로 비교합니다.

- 조음 오류 위치 시각화 (예: [f] 발음 시 아랫입술 접촉 압력 결여)
- 성조 물리적 상승·하강 곡선 대조
- 정량 백분율 점수 제공

### 5.2 크롬 익스텐션 실시간 웹 문맥 주입

외신 뉴스(NYT, CNN) 또는 바이두 웹페이지에서 단어·문장을 드래그하면:

1. 백엔드 파서가 즉시 한글 표음 변환 수행
2. 원문 위에 `<ruby>` 오버레이 투사
3. 드래그한 어휘가 SM-2 복습 큐에 자동 등록 → 1일·6일·31일 후 리마인드

### 5.3 LLM 기반 개인화 문맥 단어장

$EF$ 수준이 낮은 취약 단어를 자동 선별 → Gemini API로 사용자의 개인 관심사(IT, 패션, SF 등)와 결합된 3문장 시나리오를 생성합니다.

- 생성된 문장에 한글 표음 루비 + TTS 오디오 자동 결합
- 기계적 암기 탈피, 정서적 의미 결합(Elaborative Rehearsal)으로 장기 기억 보존율 향상

## 6. 결론

UNewLanguage는 한글의 표음적 정밀성과 현대 멀티모달 AI 엔지니어링을 결합한 에듀테크 플랫폼입니다.

- 특허 회피: HTML5 `<ruby>` 태그 + 웹 안전 한글 자모 매핑으로 KR20130122437A와 기술적으로 완전 분리
- 비용 최적화: Gemini 2.5 Flash TTS Batch API + Redis/CDN 캐싱으로 무료 트래픽 폭증에도 수익성 유지
- 성장 전략: 무료 바이럴 → Pro 구독(MRR) → 학원 SaaS(ARR) 순차 전환

> 한글 문화권을 거점으로 영어·중국어 학습 혁신 플랫폼으로의 조기 포지셔닝을 목표로 합니다.
